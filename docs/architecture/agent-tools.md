# Agent 与 Tool 调用链

Agent 负责决策与执行编排，核心实现位于 `src/agent/loop.py`。

## AgentLoop 职责

- 消息消费与会话状态维护
- 上下文构建与 LLM 调用重试
- 工具调用解析、并行执行、结果汇总
- 预算统计、错误分类与降级处理

## 关键机制

### 1) 重试与预算

- LLM 和 Tool 拥有独立重试策略。
- 通过 usage tracker 汇总 token/cost 使用情况。

### 2) 工具安全门禁

- 高风险工具受安全等级与策略限制。
- Release gate 可对高风险动作硬阻断。

### 3) 并发调用

- 支持并行执行多个工具调用。
- 单工具失败不应导致整个请求链路崩溃，需可恢复或降级。

## ToolRegistry 职责（`src/tools/registry.py`）

- 注册/注销工具
- 计算有效策略（allow/deny/group/tier/provider）
- 在执行前评估访问权限
- 输出审计信息与拒绝原因

## 开发建议

- 新工具必须提供明确参数 schema 与安全级别。
- 复杂工具优先拆分为“计划工具 + 执行工具”。
- 工具执行结果应结构化，便于日志与前端展示。
