# Memory 与 Soul

`src/memory` 与 `src/soul` 共同构成“长期稳定 + 个性一致”的认知基础。

## Memory（长期记忆）

主要关注：

- 会话/事件持久化
- 语义索引与检索
- 回忆触发（时间、实体、语义）

工程要求：

- 写入链路可恢复，不因单次故障破坏整体索引。
- 召回结果可追踪（来源、时间、相关度）。

## Soul（人格与认知）

主要关注：

- 人格边界与风格一致性
- 工作记忆组织
- 多步思考与反思流程

工程要求：

- 人格策略可配置，不依赖硬编码提示词散落。
- 对话风格与安全边界不冲突，冲突时以安全策略优先。

## 两者协同

- Memory 提供事实与历史。
- Soul 提供行为方式与表达规则。
- Agent 在推理时同时消费两类上下文，避免“有记忆没人格”或“有人格没事实”。
